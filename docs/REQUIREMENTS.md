# 小馆点餐系统 · 需求文档

## 1. 项目概述

| 项 | 说明 |
| --- | --- |
| 项目名称 | 小馆点餐系统 |
| 技术栈 | 后端 Go + Gin + GORM + SQLite；前端 Vue 3 + Vite |
| 目标用户 | 小型餐饮场景下的点餐与后厨管理 |
| 部署形态 | 本地单机运行，前后端分离 |

## 2. 功能需求

### 2.1 点餐（FR-01）

- 仅展示**已上架**的菜品与汤品
- 点击菜品卡片可查看：图片、类型、分类、描述、**原材料及用量**、**烹饪方式**
- 支持加入购物车、调整数量、填写备注并下单
- 下单时按菜品配方**自动扣减**原材料库存；无限库存项不扣减
- **不含价格**，购物车与订单以**件数**统计

### 2.2 随机轮盘（FR-02）

- 轮盘扇区与**上架菜品列表**一一对应，列表变化时自动更新
- 至少 2 道上架菜品方可转动
- 转动过程：扇区高亮 + 音效（可静音）
- 抽中后：自动加入购物车，并跳转至点餐页

### 2.3 菜品 / 汤品管理（FR-03）

- 支持新增、编辑、删除、上架 / 下架
- 类型：**菜品** / **汤品**
- 图片：拍照、上传或粘贴 URL
- 烹饪方法：手动填写；或通过 **AI 推荐**；或通过 **来源网站 URL 抓取提炼**
- 原材料：**手动关联**已有项，或**新建原材料**后关联（每份用量自填）

### 2.4 原材料 / 库存（FR-04）

- 原材料 CRUD
- 库存可为具体数量或**无限**（`stock = -1`）
- 无限库存不参与下单扣减

### 2.5 订单记录（FR-05）

- 查看历史订单及明细
- 状态：待处理 → 已支付 → 完成；支持取消

### 2.6 项目文档（FR-06）

- 系统内提供可视化文档入口
- 包含：本需求文档、更新日志、REST 接口文档及在线调试

## 3. 非功能需求

| 编号 | 要求 |
| --- | --- |
| NFR-01 | API 密钥等敏感信息**不得**写入代码库，仅通过环境变量配置 |
| NFR-02 | 前端开发环境通过 Vite 代理访问 `/api` 与 `/uploads` |
| NFR-03 | 数据库使用 SQLite，首次启动自动迁移并写入演示数据 |
| NFR-04 | AI 接口兼容 OpenAI 风格 `chat/completions` |

## 4. 数据模型（概要）

- **Ingredient**：名称、单位、库存（-1 为无限）
- **Dish**：名称、描述、分类、类型（dish/soup）、烹饪方法、图片、上架状态
- **DishIngredient**：菜品与原材料的多对多及每份用量
- **Order / OrderItem**：订单头与明细（无价格字段）

## 5. 接口约定

- 基础路径：`/api`
- 统一错误：`{ "error": "消息" }`
- 详细接口见系统内「项目文档 → 接口文档」或仓库 `README.md`

## 6. 环境变量

| 变量 | 必填 | 说明 |
| --- | --- | --- |
| `AI_API_KEY` | AI 功能时必填 | 大模型 API Key |
| `AI_BASE_URL` | 否 | 默认 `https://api.openai.com/v1` |
| `AI_MODEL` | 否 | 默认 `gpt-4o-mini` |

参考 `backend/.env.example` 配置本地环境。
